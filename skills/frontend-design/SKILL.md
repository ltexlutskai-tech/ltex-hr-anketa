# Frontend Design Skill

## Design Thinking Process
Before coding, commit to a BOLD aesthetic direction:
1. **Purpose**: What problem does this interface solve? Who uses it?
2. **Tone**: Choose deliberately: brutally minimal / maximalist / retro-futuristic / luxury / playful / brutalist / editorial
3. **Differentiation**: What makes this UNFORGETTABLE?

## Aesthetic Guidelines

### Typography
- Choose DISTINCTIVE fonts — avoid Arial, Inter, Roboto, system fonts
- Pair display font + refined body font
- Use Google Fonts or system stacks creatively

### Color & Theme
- Use CSS variables for all colors
- Dominant colors + sharp accents (avoid timid even palettes)
- Commit fully — no wishy-washy gradients

### Motion & Interaction
- CSS animations for page load (staggered reveals with animation-delay)
- Hover states that surprise
- One well-orchestrated entrance beats scattered micro-interactions

### Spatial Composition
- Unexpected layouts, asymmetry, overlap
- Generous negative space OR controlled density
- Grid-breaking elements where appropriate

### Backgrounds
- Create atmosphere: gradient meshes, noise textures, geometric patterns
- Dramatic shadows, layered transparencies
- NEVER default to plain white/gray backgrounds

## Implementation Rules
- Single HTML file (CSS + JS embedded) unless told otherwise
- Mobile-first responsive design
- Production-grade, functional code
- NEVER use purple gradients on white — overdone AI cliché
- Every design should look unique, not templated

## For L-TEX Projects Specifically
- Brand feel: European quality, trustworthy, commercial B2B
- Colors to consider: deep navy, warm white, amber/gold accent
- Tone: professional yet approachable, not corporate-sterile
- Typography: something with character but readable at small sizes
